由于理论部分介绍过，syscall执行时会将下一条指令RIP保存到寄存器中，因此syscall处理完系统调用后就会执行cmp指令。 cmp指令做的事情是将%rax寄存器的值(系统调用的返回值)与0xfffffffffffff000（-4096）进行比较 如果返回值比-4096大（JA的意思是Jump if Above，无符号大于），则跳转到0x133c30处继续执行逻辑 可以看出从0x133c30开始是对errno的赋值:
1. 将全局的一个变量值读入%rdx
2. 将%rdx的值取反（neg）
3. 为该线程的errno变量赋值（在 x86-64 Linux 中，FS 寄存器用于线程局部存储 (Thread Local Storage, TLS)）
4. __write()返回值(%rax)设置为0xffffffffffffffff（-1）(函数传参章节介绍过，%rax是被调用者返回值使用) 因此从机器码中，我们也可以验证之前理论部分学到的内容：
5. syscall的执行与返回, write的系统调用号为0x1
6. errno是一个线程局部变量
7. errno的赋值原理

### 为什么是-4096？
为什么系统调用的返回值是和-4096比较，而不是和0比较呢？

原因是Linux定义的错误码的取值范围是[1-4095]，而内核返回的是错误码的负值。又因为负数在寄存器中都是以补码的形式存在，例如0xfffffffffffff000是有符号的-4096，而0xfffffffffffff001是有符号的-4095,而0xffffffffffffffff是有符号的-1。那么机器码的意思就很明确了，如果返回值是负数，则检查无符号情况下是否比-4096大。

由于假如执行成功，则返回0或者一个正数，在无符号的情况下都不会比最高位为1（符号位）的负数来得大。

通过查看Linux内核源码可以确认这一段机器码的来源。
```C
#define MAX_ERRNO	4095

#define IS_ERR_VALUE(x) unlikely((x) >= (unsigned long)-MAX_ERRNO)
```

strace是设计用来跟踪进程的系统调用和信号的工具。

可以使用strace <程序名>启动并持续追踪程序的系统调用，也可以通过strace -p <pid>的方式监听一个已经运行的程序的系统调用，下面将给出这两种的例子

